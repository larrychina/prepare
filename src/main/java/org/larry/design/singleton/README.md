### 单例模式

#### 饿汉式  
 1，定义时实例化  
 2，静态代码块实例化  
优点：
    无需考虑线程安全问题      
缺点：不适用单例对象较多的场景，在服务启动时，需要一次把所有实例都加载到内存中，暂用资源

#### 懒汉式
1，使用时实例化，方法上加同步锁，不推荐,大量线程阻塞  
2，使用时实例化，double check ,变量使用volatitle 修饰  
3，静态内部类，对上面进行完美优化

优点：
    懒加载  
缺点：  
    1，线程安全问题  
    2，利用反射可以破坏单例模式  
    3，反序列化可以破坏单例模式
    
解决反射破坏单例模式：     
对构造方法增加判空，如果存在则报错

解决序列化和反序列化破坏单例模式：

``readResolve()``方法

    ``return INSTANCE``
